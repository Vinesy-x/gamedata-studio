# GameData Studio v1.0 — 产品方案

> **版本**：v1.0 | 2026-03-11（与代码同步更新）
> **产品类型**：Excel Web Add-in（侧边栏加载项）
> **目标用户**：RPG/MMO 游戏数值策划团队

---

## 一、产品定位

GameData Studio 是一个 Excel 侧边栏加载项，为游戏数值策划团队提供数据导出、校验和 Git 版本管理能力。完全替代现有 VBA 宏，保留 Excel 原生编辑体验。

### 1.1 核心价值

- 保留 Excel 原生编辑能力，策划无需学习新工具
- 侧边栏面板提供导出、校验、Git 等增强功能
- 精确的错误码机制，快速定位数据问题
- 支持版本筛选和地区线路筛选，一键导出

### 1.2 v1.0 范围

| 功能模块 | 说明 | 状态 |
|---------|------|------|
| 导出引擎 | 版本筛选 + 差异对比 + 文件写入 | 已完成 |
| 数据校验 | Excel错误值、空格、版本格式校验 | 已完成 |
| Git 集成 | 生成推送命令复制到剪贴板 | 已完成 |
| 侧边栏 UI | 4个tab页签 + 内联结果展示 | 已完成 |
| 网页协同导出 | OneDrive 同步方式 | 暂未实现 |
| 预览功能 | 导出数据预览 | 占位中 |
| 独立校验 | 不导出仅校验 | 占位中 |

## 三、核心功能流程

### 3.1 导出流程

```
用户操作                        系统行为
────────                        ────────
打开 Excel 侧边栏           →  加载配置，显示当前版本/版本号
点击「开始导出」             →  10步导出流程：
                                 1. 加载配置
                                 2. 初始化筛选器
                                 3. 加载所有数据表到内存
                                 4. 数据校验
                                 5. 加载已有全部表（差异对比）
                                 6-8. 逐表筛选+对比+写入
                                 9. 保存全部表
                                 10. 更新状态
查看内联结果                 →  摘要+文件列表+警告+错误
点击 [定位] 跳转修正        →  Excel 跳转到出错单元格
点击「Git」按钮             →  复制推送命令到剪贴板
在终端粘贴执行               →  Git add + commit + push
```

### 3.2 Git 上传流程

```
v1.0 实现方式：
1. 用户在导出页点击「Git」按钮
2. 系统生成完整的 Git 推送脚本：
   cd "输出目录" && git add "file1.xlsx" && ... && git commit -m "提交信息" && git push
3. 脚本复制到剪贴板
4. 弹出提示："Git 推送命令已复制到剪贴板，请在终端中粘贴执行。"
5. 用户打开终端粘贴执行

v2.0 计划：接入 companion server 实现自动执行。
```

### 3.3 数据校验

导出前自动执行校验，不中断导出，结果以警告形式展示：

| 校验项 | 说明 | 错误码 |
|--------|------|--------|
| Excel 错误值 | 检测 #N/A, #REF! 等 | 2203 |
| 纯空格单元格 | 有内容但 trim 为空 | 2204 |
| 版本格式：横线 | 使用 - 而非 ~ | 2101 |
| 版本格式：无效 | 数字格式不合法 | 1003 |
| 版本格式：倒置 | min > max | 2103 |
| 版本格式：双空 | 波浪号两侧同时为空 | 2104 |

配置区域（versionRowData/versionColData）中的空值是合法的，不触发告警。
主数据区的检测范围为 Ctrl+A 有效区域（连续非空字段列 x 首列非空的行）。

## 四、技术约束

### 4.1 文件系统

Office Web Add-in 运行在浏览器沙箱中，无法直接访问文件系统。
v1.0 通过本地开发服务器的 API 端点实现文件读写：
- `POST /api/write-file` — 写入文件（base64 编码的 ArrayBuffer）
- `GET /api/read-file` — 读取文件

### 4.2 Git 执行

同样因为沙箱限制，v1.0 不直接执行 Git 命令，而是生成命令脚本复制到剪贴板。

### 4.3 Excel 数据读取

使用 Office JS API 的 `getUsedRange()` 一次性读取工作表数据到内存（SheetSnapshot），
所有解析和筛选操作在内存中完成，避免频繁的 API 调用。

注意：UsedRange 可能不从 A1 开始，SheetSnapshot 记录 startRow/startCol 偏移量，
写回工作表时需加上偏移。

### 4.4 无 version_r 的表

部分辅助表（如技能buff表）没有 version_r 标记，DataLoader 返回 null，不参与导出。
这是正常行为，不报错。

## 五、网页协同导出

v1.0 暂未实现网页协同导出功能。原 VBA 通过检测 OneDrive 同步状态来协调多人导出，
Add-in 版本的协同方案需要进一步设计，计划在后续版本中实现。
